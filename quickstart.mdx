## Overview

Use this quickstart to create a new WordPress theme powered by TAW Theme and TAW Core, run the Vite dev server, and scaffold your first block.

<Callout kind="info" collapsed="false">
  TAW Theme uses TAW Core under the hood for the block system, MetaBlock, metabox engine, and Vite asset pipeline. When you boot the theme, you also boot TAW Core.
</Callout>

## Prerequisites

<Callout kind="alert" collapsed="false">
  Before you start, make sure your environment meets these requirements:

  - WordPress 6.0 or higher

  - PHP 8.1 or higher

  - Composer installed and available on your `PATH`

  - Node.js 20 or higher

  - npm (bundled with Node)
</Callout>

## Create and activate a TAW Theme project

<Steps>
  <Step title="Create a new theme via Composer" icon="terminal" title-type="p">
    Run the Composer template to scaffold a fresh TAW Theme project inside your WordPress `wp-content/themes` directory.

    <CodeGroup show-lines="true" tabs={[]}>
      ```bash
      # From your WordPress root
      cd wp-content/themes

      # Scaffold a new TAW Theme project (replace acme-docs with your theme folder)
      composer create-project taw/theme acme-docs

      cd acme-docs

      # Install PHP dependencies for TAW Core, Vite integration, and block system
      composer install
      ```

      ```bash
      # If you already created the project, just install PHP deps
      cd wp-content/themes/acme-docs
      composer install
      ```
    </CodeGroup>

    After this step, your theme folder contains TAW Theme, TAW Core, and the `Blocks` directory for your custom blocks.
  </Step>

  <Step title="Install JS dependencies and start Vite dev server" icon="zap" title-type="p">
    Install the Node dependencies and start the Vite dev server for hot module reloading during development.

    ```bash
    # Still inside your theme directory
    npm install

    # Start Vite with HMR for Tailwind v4 and your block assets
    npm run dev
    ```

    When this succeeds, your terminal shows Vite listening on a localhost port, and WordPress pages using the theme will load assets from the dev server.
  </Step>

  <Step title="Activate the theme in WordPress admin" icon="monitor" title-type="p">
    Switch your site to use the new TAW Theme.

    - Log in to your WordPress admin.

    - Go to **Appearance → Themes**.

    - Find your new theme (for example, **Acme Docs**).

    - Click **Activate**.

    After activation, visit the frontend: you should see the TAW Theme layout rendered, backed by TAW Core.
  </Step>
</Steps>

## Scaffold your first block

Use the built-in TAW CLI to generate a new block and place it in the `Blocks` directory.

<Steps>
  <Step title="Generate a block with the TAW CLI" icon="code" title-type="p">
    Run the block generator from your theme root.

    ```bash
    # From your TAW Theme directory
    php bin/taw make:block Hero --type=meta --with-style
    ```

    This command creates a new `Hero` block inside `Blocks/`, including PHP, meta configuration, and style scaffolding wired into the Vite pipeline.
  </Step>

  <Step title="Verify the block in WordPress" icon="check-circle" title-type="p">
    After generation and with `npm run dev` still running:

    - Go to the WordPress block editor for a page or template.

    - Search for the **Hero** block in the block inserter.

    - Insert it and confirm that its markup and styles load correctly.

    If you see the block and edits update live when you save block files, your TAW Theme + TAW Core setup is working.
  </Step>
</Steps>

## Optional: production build

Use a production build when you deploy or want to test optimized assets.

```bash
# From your TAW Theme directory
npm run build
```

This runs the Vite production build and outputs optimized CSS and JS that WordPress will load when the dev server is not running.
