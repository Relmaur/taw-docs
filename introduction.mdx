## Start here (5–10 minutes)

TAW is a modern, component-based WordPress framework built around PHP blocks, MetaBlocks, and a Vite/Tailwind asset pipeline.

<Callout kind="info" collapsed="false">
  TAW Theme is designed for WordPress and PHP developers who prefer code-first workflows, version control, and repeatable architecture over click-driven theme builders.
</Callout>

<Callout kind="alert" collapsed="false">
  This is an early draft of the TAW documentation. The framework is fully code-first: expect to work in PHP, Composer, and Node, with minimal WordPress admin configuration.
</Callout>

Use these pages to get oriented:

- Start with [Quickstart](/quickstart) to install TAW Theme via Composer, wire up Vite, and see your first block render.

- Read [Core Concepts](/concepts) for a deeper look at blocks, MetaBlocks, assets, and how TAW Theme and TAW Core interact.

- Browse [Components](/components) when you want concrete examples of how we structure code, blocks, and documentation.

## What you get

TAW Theme depends on the `taw/core` Composer package (TAW Core) for block discovery, the MetaBlock engine, Vite/Tailwind integration, and helper utilities.

<Columns cols="3">
  <Card title="Component and block architecture" href="/concepts" icon="layers" horizontal="false">
    Organize your theme into PHP blocks under `Blocks/`, with automatic discovery and loading via TAW Core. Build UI blocks and MetaBlocks that stay close to your domain instead of scattered templates.
  </Card>

  <Card title="MetaBlocks and config-driven data" href="/concepts" icon="database" horizontal="false">
    Define MetaBlocks backed by a metabox engine using `post_meta` and options pages. Describe fields and layouts in PHP config so metaboxes and settings screens are reproducible and versioned.
  </Card>

  <Card title="Modern asset pipeline" href="/quickstart" icon="terminal" horizontal="false">
    Use Vite for bundling with HMR in development and Tailwind CSS v4 via `@tailwindcss/vite`. Scope assets per block, keep CSS close to components, and load everything through TAW Core’s asset helpers.
  </Card>
</Columns>

## Make your first TAW theme

Use this flow to go from a blank project to a running TAW Theme backed by TAW Core.

<Steps>
  <Step title="Install the theme and TAW Core" icon="download" title-type="p">
    Initialize a new WordPress theme with the TAW Theme template via Composer, run `composer install` so `taw/core` is available, then wire up `functions.php` to require the autoloader and call `TAW\Core\Theme::boot()`.
  </Step>

  <Step title="Set up Vite and Tailwind v4" icon="terminal" title-type="p">
    Run `npm install`, configure the provided Vite setup (including `@tailwindcss/vite`), and start the dev server with `npm run dev` to get hot module reloading and Tailwind utilities in both frontend and editor.
  </Step>

  <Step title="Create your first Block and MetaBlock" icon="edit" title-type="p">
    Use the TAW CLI (for example, `php bin/taw make:block ...`) to scaffold a new block under `Blocks/`, define a MetaBlock for structured content, and verify it appears in the editor with its own assets and metabox-backed fields.
  </Step>

  <Step title="Explore Core Concepts and patterns" icon="book-open" title-type="p">
    Once the first block works, read [Core Concepts](/concepts) to standardize how you structure `Blocks/`, design MetaBlocks, and configure options pages so future themes reuse the same patterns.
  </Step>
</Steps>
